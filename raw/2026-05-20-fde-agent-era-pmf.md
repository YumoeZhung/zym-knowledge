# OpenAI、Anthropic 都开始押注 FDE，FDE 才是 Agent 时代的 PMF 范式？

Agent时代的第一个共识浮出水面，模型已经不是瓶颈，人才是。

短短一周，OpenAI砸下40亿美元、Anthropic嵌进FIS的总部、Google官宣招几百人——三家AI巨头不约而同押注的，是同一个职位：FDE。

## 背景事件

- 2026年5月11日，OpenAI宣布成立 OpenAI Deployment Company，首期投入40亿美元
- Anthropic把工程师团队嵌进金融科技巨头FIS，目标让反洗钱调查从几小时压缩到几分钟
- Google Cloud CEO Thomas Kurian招聘几百名FDE，消息获得130万次浏览

## 一、FDE 是什么

FDE（Forward Deployed Engineer）由Palantir在2000年代初发明，当时客户是美国情报机构。

Palantir联合创始人Shyam Sankar：如果一个问题能被需求文档解决，那它早就被解决了。

做法：把工程师直接派进客户的办公室、军事基地、飞机总装车间。内部称Delta。

与传统角色的区别：
- 销售工程师：售前演示，签合同就走
- 解决方案架构师：技术咨询，不写生产代码
- 咨询顾问：出方法论，不参与产品迭代

FDE特殊性：写客户的生产代码，同时把共性问题反哺回厂商核心产品。

到2016年Palantir的FDE数量超过普通软件工程师。Foundry平台就是从FDE驻场项目中提炼出来的。

模式总结：砂石路到柏油路——FDE在客户现场修砂石路，产品团队找出走的人最多的几条铺成柏油路变成平台能力。

## 二、2026年的拐点

核心判断：Agent时代的瓶颈不在模型本身而在部署。

数据：
- 埃森哲调研：只有32%企业领导者看到持续的企业范围AI影响
- 68%停留在试点/PPT/Demo阶段
- OpenAI 2026年初未完成内部营收和周活目标

核心洞察：AI模型产品力边际收益下降，把模型变成可用系统的工程能力边际收益飙升。

## 三、Agent时代对FDE的结构性需求

Agent与SaaS的根本区别：SaaS是工具（买来使用），Agent是代劳（替你做）。

三个后果：
1. Agent必须深度嵌入客户真实工作流
2. Agent失败是业务失败而非功能失败
3. Agent市场既无成熟产品可对标、客户也不知道要什么

Anthropic FDE职位描述：在客户系统内部构建生产应用、交付MCP服务器、sub-agent、agent skills等技术产物、识别可复用的部署模式并反哺回产品和工程团队。

## 四、三个保留意见

1. FDE可能在掩盖PMF问题（Gartner预测2028年70%企业将放弃FDE主导的Agent项目）
2. 伪装成产品公司的咨询公司（影响毛利和估值）
3. FDE可能被自己创造的工具替代（低层级集成被AI吃掉，核心价值转向业务判断）

## 五、对不同角色的意义

- 对AI厂商：FDE不是收入业务，是产品发现机制
- 对企业客户：真正价值是能力转移，没有退出机制就是温柔的锁定
- 对工程师：2026年最稀缺技能组合，Google FDE薪酬12.7万~26.5万美元基薪，顶级接近40万
- 对投资人：看砂石路变柏油路速度，每次驻场后产品可复用能力提升多少

## 结论

FDE是Agent时代企业级AI从Demo走向生产系统的必要中间态，但它本身不是PMF——它是寻找PMF的方法。
